Food Report - це веб-аплікація по роботі з звітами по продуктам. Ціль - зберігати замовлення і продукти. Сервіс виконує вибірку, внесення нових даних, маніпуляція з ним, виведення по фільтрам та підсумкових звітів. 

Проект створюється на Asp.net Core 2.0. Для зберігання данних використовується mongoDB. Для роботи з базою данних використовуються патерни Unit Of Work і Repository.
Також використовується Javascript для побудови таблиць на стороні клієнта і відправки AJAX запитів. Для хешування паролів користувачів використовується кастомний сервіс з викоористанням бібліотеки Microsoft.AspNetCore.Cryptography.KeyDerivation

Для роботи потрібно:
1. Запустити VS 2017 і запустити проект FoodReport. Для входу на сайт вам потрібно авторизуватись, для цього можна використати профіль адміністратора (логін admin@ad, пароль admin)
2. Перейти на вкладку Product і створити там декілька позицій. Після цього, на вкладці Report можна створити звіт з використанням тих продуктів, які є на сторінці Product.

Реалізовані CRUD операції з продуктами, звітами, ролями та користувачами.
Користувач з правами адміністратора має доступ до адмінки, де можна створити нового користувача, змінити існуючому користувачу пароль, змінити роль(права) і т.д. Також адміністратор може створювати нові позиції продуктів, редагувати їх та видаляти.
На сторінці звітів, всі користувачі можуть переглядати  та створювати звіти, редагувати їх можуть лиш їх творці, а видаляти лиш адміністратори. Також адміністратор може підтвердити звіт, або його відхилити.
Реалізований пошук по продуктах (за назвою або постачальником) та звітах (за датою, творцем чи статусом).
Розроблений сервіс підсумкових звітів, які будуть формуватись за певний період (за дату, місяць чи рік) з звітів, які є підтверджені адміністратором. Присутня функція створення підсумкового звіту за певний термін. Після створення декількох звітів та їх підтвердження можна обрати часові рамки та натиснути на кнопку "Summary" на сторінці звітів, і має з'явитись сторінка з таблицею, де вказані всі продукти з звітів, їх кількість ( якщо в декількох звітах є один і той же товар, то в підсумковому звіті їх кількість сумується) та вартість, після таблиці підбиті підсумки у вигляді сумарної кількості позицій товару та загальна ціна.
